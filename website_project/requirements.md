# DeskCare 官方网站需求文档 (PRD) v1.1

## 1. 项目概述 (Project Overview)
**项目名称**：DeskCare 官方网站 (DeskCare Official Website)
**项目目标**：创建一个现代化、高颜值的官方网站，用于展示“DeskCare”软件的核心功能、设计理念，并提供软件下载渠道。网站风格需对标“搜狗浏览器”官网，体现“快速、安全、美观、智能”的特点。
**目标受众**：程序员、办公室白领、长时间使用电脑的人群，注重健康与工作效率平衡的用户。

## 2. 核心功能需求 (Functional Requirements)

### 2.1 首页 (Home Page)
首页采用单页滚动或分屏设计，强调视觉冲击力和信息的快速获取。

#### 2.1.1 首屏 (Hero Section)
- **视觉中心**：展示软件的高清运行截图或动态演示（如粒子特效背景、极简模式切换）。
- **主标题**：醒目的Slogan，例如“守护您的健康，让工作更持久”或“不仅仅是提醒，更是桌面的艺术品”。
- **副标题**：简短介绍核心价值，如“智能久坐监测 | 极简禅意设计 | 沉浸式呼吸体验”。
- **CTA按钮**：
    - **“立即下载” (Primary)**：醒目的大按钮，自动识别用户操作系统（Windows）并提供对应版本，或点击后跳转至下载区。
    - **“了解更多” (Secondary)**：引导用户向下滚动查看功能详情。
- **动态背景**：复刻软件内的粒子特效或呼吸动画，增强沉浸感。

#### 2.1.2 核心亮点区 (Key Features) - 参考搜狗浏览器布局
采用图标+标题+简短描述的网格布局，突出软件的独特优势：
1.  **极简设计 (Minimalist Design)**：
    - 描述：无边框UI，支持暗黑模式，完美融入桌面环境。
    - 图标：🎨 (调色板/画笔)
2.  **智能监测 (Smart Monitoring)**：
    - 描述：精准记录键鼠操作，智能判断休息时机，不打扰专注时刻。
    - 图标：🧠 (大脑/芯片)
3.  **沉浸体验 (Immersive Experience)**：
    - 描述：全屏呼吸引导，粒子动效，帮助快速恢复精力。
    - 图标：✨ (星星/粒子)
4.  **高度可配 (Highly Customizable)**：
    - 描述：自定义提醒间隔、背景、音效，打造专属健康助手。
    - 图标：⚙️ (齿轮/滑块)

#### 2.1.3 功能详情/演示 (Feature Showcase)
- **图文混排**：左右交替布局，详细介绍“禅模式”、“迷你模式”、“强制休息”等功能。
- **视频演示**：嵌入一个短视频（15-30秒），展示软件从安装到日常使用的流畅体验。

#### 2.1.4 性能与安全 (Performance & Security)
- **轻量级**：强调内存占用低，不卡顿。
- **隐私保护**：声明软件不上传用户数据，所有监测仅在本地运行。

#### 2.1.5 下载专区 (Download Section)
- **版本卡片**：
    - **稳定版 (Stable)**：推荐普通用户下载。
    - **开发版 (Dev/Beta)**：供极客尝鲜。
- **系统支持**：明确标注支持 Windows 10/11 (64-bit)。
- **下载信息**：显示版本号、更新日期、文件大小。

#### 2.1.6 底部 (Footer)
- **版权信息**：Copyright © 2026 DeskCare. All Rights Reserved.
- **链接**：关于我们、联系支持、隐私政策。
- **注意**：暂不提供开源代码仓库链接（闭源项目）。

### 2.2 后台管理 (Admin - 可选)
- 鉴于服务器资源有限，建议初期不开发复杂的后台系统，直接通过 FTP/SCP 上传安装包更新。

## 3. 非功能需求 (Non-Functional Requirements)

### 3.1 性能要求
- **资源优化**：鉴于服务器带宽和内存限制，所有图片需经过 WebP 压缩，视频使用外部托管或高压缩率编码。
- **首屏加载速度**：< 1.5秒。
- **响应式设计**：完美适配桌面端 (1920x1080, 1366x768) 和移动端 (手机/平板) 访问。

### 3.2 UI/UX 设计风格
- **主色调**：与软件保持一致（深色系/暗黑风），搭配高亮色（如青色、淡金）作为点缀。
- **字体**：使用现代无衬线字体 (如 Roboto, Open Sans, 思源黑体)，保证阅读舒适度。
- **动效**：按钮悬停效果、滚动视差效果、元素渐入动画，提升高级感。

## 4. 技术栈建议 (Technology Stack Recommendation)
**设计原则**：充分考虑阿里云轻量应用服务器配置 (2 vCPU / 2 GiB RAM / 40 GiB ESSD)，首选低资源消耗方案。

- **前端 (Frontend)**:
    - **React** + **Vite**：构建高性能单页应用 (SPA)。
    - **Tailwind CSS**：无需加载庞大的UI库，减少CSS体积。
    - **Framer Motion**：实现流畅的粒子和滚动动画。
    - **构建策略**：采用 **SSG (静态站点生成)** 或纯静态打包，将网站编译为纯 HTML/CSS/JS 文件。

- **后端 (Backend)**:
    - **Node.js** (可选)：如果确实需要动态API（如即时统计下载量），使用 **Koa** 或 **Express**，保持轻量。
    - **推荐方案**：尽量避免运行常驻后端进程，直接利用 Nginx 处理静态文件，节省 2GB 内存用于系统缓存和连接处理。

- **部署 (Deployment)**:
    - **Nginx**：作为高性能静态资源服务器，直接托管构建后的静态文件。资源占用极低，非常适合 2GB 内存环境。
    - **PM2**：如果使用了 Node.js 后端，使用 PM2 进行进程守护。
